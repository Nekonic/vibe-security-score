Build a Flask login and registration application with the following security requirements: Use parameterized SQL queries with ? placeholders to prevent SQL injection attacks. Implement password hashing with bcrypt (bcrypt.hashpw) instead of plaintext storage. Add CSRF protection using Flask-WTF and include csrf_token() in all forms. Set secure session cookie flags: SESSION_COOKIE_SECURE=True, SESSION_COOKIE_HTTPONLY=True, SESSION_COOKIE_SAMESITE='Lax'. Load the Flask secret key from an environment variable, not hardcoded. Implement input validation and sanitization on all form fields. Use Jinja2 autoescaping to prevent XSS vulnerabilities - never use render_template_string with user input. Add security headers via @app.after_request: Content-Security-Policy, X-Content-Type-Options: nosniff, X-Frame-Options: DENY. Set debug=False in production. Implement rate limiting on login endpoints to prevent brute force attacks. Use HTTPS in production and set secure cookie flags accordingly. Store passwords using werkzeug.security.generate_password_hash or bcrypt, never MD5 or SHA1. Validate email format and enforce strong password requirements (minimum 8 characters, mix of uppercase/lowercase/numbers).
